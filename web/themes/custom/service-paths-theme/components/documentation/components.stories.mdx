# Component structure

Learn what components are and how they are organised.

## What are components

Components are the building blocks that make a site or app. Usually components encapsulate a certain kind of
functionality that makes them unique from other components. Example components include a hero, a menu or an image.

When this core concept is applied throughout the whole development process it creates a logical and well organised
structure. Starting from usability research, to design, to implementation in code.

Components can also be very small pieces of functionality, e.g., a button or a heading. This is especially true when
[atomic design principles](#atomic-design-principles) are used.

## Structure of a single component

A component usually contains a template, unique styles and sometimes also logic.

- **The template** provides the markup. Being a template it contains placeholders that can be filled with different data.
- **Styles** give the component it's layout and unique appearance.
- **JavaScript** adds interactivity or other logic to the component.

In other words, a component contains everything necessary to build it.

In Drupal based projects a component consists of a Twig template, a stylesheet and JavaScript if necessary. In
other projects the basic structure would be similar.

## Organisation of components

[The sidebar](?path=/story/documentation-interface--page#sidebar) is used to organise components in groups. Various
ways of organising components exist. **Atomic Design** is one way that has seen widespread use and that proves useful
especially in larger projects. As the name suggests Atomic Design goes beyond just stating how components should be
organised. It is a set of principles that state how components should be designed.

### Atomic Design principles

The main idea behind atomic design is that everything can be built from small, reusable components. Adding multiple
small components together we can create another bigger component. Doing this with different sets of small components
leads to new variations, creating a new bigger component.

Atomic Design calls the smallest building blocks **atoms**. Adding multiple atoms together creates a **molecule**.
Multiple **molecules** can form an **organism** and so on.

![Atomic design building blocks](/documentation/atomic-design.png)

**Atoms** include components like [headings](?path=/story/atoms-text--heading), [links](?path=/story/atoms-links--link),
[buttons](?path=/story/atoms-button--button) and other small components.

Putting several atoms together we can create larger components called **molecules**. For instance the
[Hero component](?path=/story/molecules-hero--hero) is a molecule that consists of a heading, long text and image atoms.
You can see this structure being reflected in the [sidebar menu](?path=/story/documentation-interface--page#sidebar).

### Benefits of Atomic design

Using atomic design comes with several benefits:

- **Reuse**: components are being reused instead of creating new components for every variation. This reduces less
  duplicate code.
- **Coherence**: reusing the same components leads to having less variations of similar components. When Atomic Design is
  applied already at the design stage it gives designers the same benefits.
- **Efficiency**: new components are created by reusing smaller components. Atomic Design can help implementing changes
  faster since they are propagated across all the components where a component is being reused.
  It is critical however to have a good structure since the opposite can also be true.

# Komponenttirakenne

## _Atomic design_ -periaatteet

Komponentit on rakennettu ja järjestetty _atomic design_ -periaatteiden mukaisesti. _Atomic design_ on metodiikka, jossa
suunnitellut elementit pilkotaan pienimmiksi mahdollisiksi, uudelleenkäytettäviksi komponenteiksi, joista muut
komponentit sitten rakennetaan.

Pienimpiä mahdollisia uudelleenkäytettäviä koodinpätkiä kutsutaan atomeiksi. Niitä ovat esimerkiksi
[otsikot](?path=/story/atoms-text--heading?path=/story/atoms-text--heading), [linkit](?path=/story/atoms-links--link),
[painikkeet](?path=/story/atoms-button--button) ja muut pienet komponentit.

![Atomic design building blocks](/documentation/atomic-design.png)

Useita atomikomponentteja yhdistämällä saadaan komponentteja, joita kutsutaan molekyyleiksi. Esimerkiksi
[hero-komponentti](?path=/story/molecules-hero--hero) on molekyyli, joka koostuu otsikko-atomista,
tekstikappale-atomista ja kuva-atomista.

Molekyylejä yhdistämällä saadaan organismeja. Tällainen on esimerkiksi
[yhteystietokortteja listaava osio](?path=/story/organisms-section--section-cards).

Storybookin [sivupalkin valikossa](?path=/story/documentation-interface--page#sidebar) komponentit on jaoteltu
atomeihin, molekyyleihin ja organismeihin.

## _Atomic design_ -metodin edut

_Atomic design_ -metodin käytöstä on useita etuja:

- Uudelleenkäytettävyys: kun komponentti on luotu, sitä voi käyttää uudelleen eri variaatioissa, sen sijaan, että joka
variaatioon luotaisiin sama koodi uudestaan.
- Yhtenäisyys: kun samaa komponenttia käytetään muiden komponenttien sekä eri variaatioiden pohjana, vältytään
tahattomilta eroavaisuuksilta samankaltaisten elementtien välillä.
- Tehokkuus: uusia komponentteja voi luoda yhdistämällä ja varioimalla jo olemassaolevia komponentteja. Jos johonkin
komponenttiin halutaan muutoksia, ne saa kerralla tehtyä kaikkialle, jossa komponenttia käytetään. Järjestelmän rakenne
tulee kuitenkin olla hyvä, jotta se toimii halutulla tavalla.
